# Завдання 1
#### Напишіть програму для визначення моменту, коли time_t закінчиться.Дослідіть, які зміни відбуваються в залежності від 32- та 64-бітної архітектури. Дослідіть сегменти виконуваного файлу.
Для того щоб визначити коли закічниться time_t ми створюємо зміну та намагаємось її переповнити,подвоюємо число так щоб кожен з 64 бітів мав значення 1,потім коли зміну вже переповнено ми отримаємо негативне число і виходимо з цикул,віднімаємо 1 від змінної і отримаємо максимально можливе число,після чого програма виведе значення цього числа,і дату.
```
Max time_t: 9223372036854775807
Date: (null)
```
Оскільки число занадто велике(приблизно 300 млрд років),то дата виводиться (null).Для того щоб команда порахуквала для 32-бітної структури,ставлю в Ubuntu multilib
```
sudo apt update
sudo apt install gcc-multilib
```
Компілюємо
```
gcc -m32 main.c -o Ex1_32
```
В результаті отримаємо
```
./Ex1_32
Max time_t: 2147483647
Date: Tue Jan 19 03:14:07 2038
```
За допомогою команди **size** отримаємо інформацію про сегменти
```
 size Ex1
   text    data     bss     dec     hex filename
   1733     616       8    2357     935 Ex1
```
Для 32-бітної:
```
size Ex1_32
   text    data     bss     dec     hex filename
   1595     312       4    1911     777 Ex1_32

```
# Завдання 2
## Розгляньте сегменти у виконуваному файлі.
#### 1.Скомпілюйте програму "hello world", запустіть ls -l для виконуваного файлу, щоб отримати його загальний розмір, і запустіть size, щоб отримати розміри сегментів всередині нього.
Компілюємо:
```
ls -l hello
-rwxrwxr-x 1 arthas arthas 15960 Mar 17 09:20 hello
```
Отримано загальний розмір файлу 15960 байтів.
Запускаємо size:
```
size hello
   text    data     bss     dec     hex filename
   1376     600       8    1984     7c0 hello
```
Отримано розмір сегментів.
#### 2.Додайте оголошення глобального масиву із 1000 int, перекомпілюйте й повторіть вимірювання. Зверніть увагу на відмінності.
Загальний розмір файлу залишився майже не змінним.
Розмір сегментів збільшився приблизно на 4000 байтів,тому що оголошено массив із 1000 елементів int,кожен з яких займає 4 байти.
```
size hello
   text    data     bss     dec     hex filename
   1376     600    4032    6008    1778 hello
```
#### 3.Тепер додайте початкове значення в оголошення масиву (пам’ятайте, що C не змушує вас вказувати значення для кожного елемента масиву в ініціалізаторі). Це перемістить масив із сегмента BSS у сегмент даних. Повторіть вимірювання. Зверніть увагу на різницю.
```
ls -l hello
-rwxrwxr-x 1 arthas arthas 20000 Mar 17 09:50 hello
size hello
   text    data     bss     dec     hex filename
   1376    4616       8    6000    1770 hello
```
bss тепер зберігає тількі службові змінні,а data значно збільшилась,тому що массив став містити ініціалізовані дані,загальний ромзмір також збільшився тому що data на відміну від bss  зберігається в ELF-файлі.
#### 4.Тепер додайте оголошення великого масиву в локальну функцію. Оголосіть другий великий локальний масив з ініціалізатором. Повторіть вимірювання. Дані розташовуються всередині функцій, залишаючись у виконуваному файлі? Яка різниця, якщо масив ініціалізований чи ні?
Без ініціалізації:
```
ls -l hello
-rwxrwxr-x 1 arthas arthas 16008 Mar 17 10:00 hello
size hello
   text    data     bss     dec     hex filename
   1543     608       8    2159     86f hello

```
З ініціалізацією:
```
ls -l hello
-rwxrwxr-x 1 arthas arthas 16008 Mar 17 11:01 hello
size hello
   text    data     bss     dec     hex filename
   1576     608       8    2192     890 hello
```
Неініцілізоловані зміні зберігаються у стеку не збільшуючи розмір файлу,а ініціалізовані збільшують сегмент text.
#### 5.Які зміни відбуваються з розмірами файлів і сегментів, якщо ви компілюєте для налагодження? 
```
gcc -g  main.c -o hello
```
#### Для максимальної оптимізації?
```
gcc -O3 main.c -o hello
```
### Проаналізуйте результати, щоб переконатися, що:
-сегмент даних зберігається у виконуваному файлі;\
-сегмент BSS не зберігається у виконуваному файлі (за винятком примітки щодо його вимог до розміру часу виконання);\
-текстовий сегмент більшою мірою піддається перевіркам оптимізації;\
-на розмір файлу a.out впливає компіляція для налагодження, але не сегменти.
